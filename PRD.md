# PRD: Speech Mirror (말거울) — AI 발화 코칭 기반 모의면접 피드백 시스템

> v2 — 음성 집중 노선. 영상/자세 분석은 P2 확장 기능으로 분리, 점수 비중 재조정, 한국어 발화 코칭 차별점 강화.

| 목적        | 이력서·직종 기반 모의면접을 진행하고, 종료 후 **음성·텍스트** 데이터를 분석해 말 속도·침묵·한국어 필러 단어·답변 내용에 대한 피드백을 제공한다. |

## 2. 제품 한 줄 정의

사용자가 이력서와 직종을 기반으로 연습/실전 면접을 진행하면, 종료 후 **자신의 답변을 다시 들으며** 말 속도·침묵·한국어 필러 단어("음·어·그·약간")·답변 내용을 정량·정성으로 확인하고 다음 답변을 개선할 수 있는 AI 발화 코칭 서비스.

---

## 3. 문제 정의 및 차별점

### 3.1 문제

면접·발표 준비자는 스스로 녹화/녹음해도 무엇이 부족한지 객관적으로 판단하기 어렵다. 특히 혼자 확인하기 힘든 것:

- 말 속도가 너무 빠르거나 느린지
- 침묵·머뭇거림이 자주 발생하는지
- "음", "어", "그", "약간" 같은 **한국어 필러 단어**가 얼마나 많은지
- 답변이 질문 의도에 맞고 구조가 명확한지

모두가 "녹화해서 피드백 받아라"라고 하지만, 정작 **그 발화를 객관적으로 분석해줄 사람이 없다.** 그 빈자리를 AI가 24시간, 부끄러움 없이 채운다.

### 3.2 차별점 (경쟁사 대비)

| 경쟁                             | 한계                            | 말거울의 차별                                           |
| -------------------------------- | ------------------------------- | ------------------------------------------------------- |
| 사람인·잡다 (국내 모의면접)      | 면접 내용·꼬리질문 중심, 유료   | **발화 전달력(속도·침묵·필러)을 핵심 지표로** 정밀 분석 |
| Yoodli·Orai (글로벌 스피치 코칭) | 영어 기준("um/uh"), 한국어 약함 | **한국어 필러("음·어·그·이제·약간")·발표체에 특화**     |
| 오프라인 스피치 학원             | 고비용·시간·장소 제약           | 무료·즉시·반복, 음성만으로 가볍게                       |

> 핵심 메시지: "Yoodli는 영어, 사람인은 면접 내용 — 우리는 **한국어 발화 코칭**으로 말하기 그 자체를 교정한다."

## 4. 제품 목표

### 4.1 핵심 목표

1. 이력서·직종 입력 → 맞춤형 면접 질문 제공
2. 연습/실전 모드 선택
3. 웹에서 답변 **오디오 녹음**(영상은 다시보기용 선택 녹화)
4. 종료 후 분석하기 실행
5. 결과 화면에서 전사·지표·피드백을 함께 확인
6. 타임라인에서 발화 구간·침묵 구간·필러 위치 확인
7. 종합 점수와 개선 포인트 제공

### 4.2 비목표 (MVP 미구현)

- 로그인/회원가입, 장기 기록 저장, 관리자 페이지, 결제
- 다중 사용자 동시 세션, 완전 실시간 피드백
- **영상/자세/시선 분석 (P2 확장으로 분리)**
- 기업별 정교한 채용 평가 모델
- **모범답안 대필 / "1분 답변 스크립트" 자동 생성** — 우리는 답변을 대신 써주는 게 아니라, 사용자가 직접 말한 발화를 교정하는 제품이다. 답변 생성형 기능은 차별점(발화 코칭)을 흐리므로 의도적으로 제외한다.

## 5. 사용자 시나리오

### 5.1 연습 모드

```
이력서 업로드 → 직종 선택 → 연습 모드 선택
→ 추천 질문 10개 생성 → 사용자가 3개 선택 → 미선택 질문 중 랜덤 1개 추가
→ 총 4개 질문으로 면접 진행(오디오 녹음)
→ 종료 후 분석하기 → 전사 + 발화 분석 리포트 확인
→ 동일 질문 1회 다시 도전
```

### 5.2 실전 모드

```
이력서 업로드 → 직종 선택 → 실전 모드 선택
→ 가벼운 시작 질문 → 공통 질문 → 이력서 기반 맞춤 질문 → 직종별 질문 → (선택)꼬리 질문
→ 종료 후 분석하기 → 종합 점수 및 개선 피드백 확인
```

## 7. MVP 범위

### 7.1 반드시 구현 (P0)

이력서 업로드 · 직종 선택 · 연습/실전 모드 · 질문 생성 · **오디오 녹음** · 분석하기 · STT 전사(필러 보존) · 다시 듣기 · 말 속도 분석 · 침묵 분석 · **한국어 필러 단어 분석** · 내용 피드백 · 개선 포인트 · 전체 점수 · 타임라인·말속도 그래프 시각화

### 7.2 가능하면 구현 (P1)

추천 질문 10개 중 3개 선택 · 미선택 랜덤 1개 추가 · 다시 도전 1회 · **영상 녹화 + 다시보기(분석 없음)**

### 7.3 후순위 (P2)

**공격형 꼬리 질문(논리 약점 탐지)** · **기업 뉴스 기반 시사형 질문(실시간 뉴스 수집)** · 질문 음성 출력(TTS) · 실전 분위기 연출 · **영상/자세/시선 분석(OpenCV·MediaPipe)**

> 시사형 질문은 실시간 뉴스 수집이 들어가 데모 표면적이 커지므로 MVP에서 제외하고, 발표 시 "확장 로드맵"으로만 제시한다.

| 면접 세션 생성     | POST   | `/interview/session`                |
| 질문 조회          | GET    | `/interview/{session_id}/questions` |
| 답변 오디오 업로드 | POST   | `/interview/answer`                 |
| 분석 실행          | POST   | `/analysis/run`                     |
| 분석 결과 조회     | GET    | `/analysis/{analysis_id}`           |
| (P2) 꼬리 질문     | POST   | `/interview/followup`               |

### 8.3 DB 미사용 정책

업로드 파일은 임시 디렉토리 저장, 분석 결과는 응답으로 즉시 반환 또는 `analysis_id`로 메모리 임시 보관. 장기 저장 필요 시 SQLite 확장.

## 9. AI 및 분석 파이프라인

```
답변 오디오
→ STT (필러 보존, 단어 타임스탬프)
→ 전사 텍스트
→ 음성 지표 분석 (말 속도·침묵·발화비율)
→ 텍스트 지표 분석 (한국어 필러·반복·구조)
→ 내용 피드백 (LLM)
→ 점수 계산
→ 결과 리포트 생성
   └─ (P2) 영상/자세 분석은 별도 선택 브랜치
```

### 9.1 STT (핵심 리스크)

- 입력: 답변 오디오 / 출력: 질문별 전사 텍스트
- **필수 검증(1단계)**: STT가 "음·어·그"를 자동 제거하지 않는지, 단어 단위 타임스탬프를 주는지 먼저 확인. 자동 제거되면 필러 분석 기능이 비므로, raw/verbose 모드 또는 무음·머뭇거림을 오디오 레벨(pydub)에서 직접 검출하는 대안 확보.
- 기본 후보: CLOVA Speech / 대체: faster-whisper

### 9.2 음성 지표

| 지표         | 산출                             |
| ------------ | -------------------------------- |
| 말 속도      | 단어 수 / 발화 시간              |
| 침묵 횟수    | 일정 시간 이상 무음 구간 카운트  |
| 가장 긴 침묵 | 무음 구간 최대 길이              |
| 발화 비율    | 전체 시간 중 실제 발화 시간 비율 |

### 9.3 텍스트 지표

| 지표             | 산출                                                     |
| ---------------- | -------------------------------------------------------- |
| 한국어 필러 단어 | 사전 정의 필러("음·어·그·이제·약간·뭔가" 등) 카운트·위치 |
| 반복 표현        | 반복 단어/구문 빈도                                      |
| 답변 길이        | 문장·단어 수                                             |
| 답변 구조        | 도입-본론-결론 또는 STAR 구조 여부                       |

### 9.4 LLM 활용 (질문 생성 · 내용 피드백) — 프롬프트 설계 원칙

사용자가 프롬프트를 몰라도 전문가 수준 결과가 나오도록, 아래 구조를 **내부 시스템 프롬프트**로 고정한다.

**(가) 질문 생성** — "면접관 시점"으로 생성하고 질문마다 평가 역량을 태깅한다.

- 입력: 이력서 요약 + 직종 + 면접 종류
- 출력(JSON): `[{ "question": "...", "competency": "문제해결력" }, ...]`
- 효과: 결과 화면에서 "이 질문이 무엇을 보는 질문인지"를 답변 피드백과 연결.

**(나) 내용 피드백** — "좋아요/별로예요"가 아니라 2축 구조로 평가한다.

- 축: ① 논리 구조(결론-근거 명확성) ② 직무 전문성(사례·구체성·직무 연관성)
- 각 축에 짧은 진단 + **개선 문장 예시 1개**를 제공. (※ 답변을 통째로 대신 써주지는 않음 — 비목표 참조)
- 점수 체계와 정합: 이 2축은 종합 점수의 "답변 내용 40%"로 매핑.

### 9.5 (P2) 자세 지표 — 확장 기능

얼굴 검출 비율 · 화면 이탈 · 자세 안정성 · 시선 안정성. _감정 단정 없이 관찰 가능한 지표만 제공._

## 10. 점수 계산 기준

### 10.1 전체 점수 (100점) — 영상 비중 제거, 발화 중심 재조정

| 항목              | 비중    | 비고                                        |
| ----------------- | ------- | ------------------------------------------- |
| 답변 내용         | 40%     | 질문 적합성·구체성·직무 연관성              |
| **말하기 전달력** | **35%** | 말 속도·침묵·**필러 단어** (제품 핵심 지표) |
| 답변 안정성       | 25%     | 답변 흐름·반복 표현·과도한 공백             |

> 기존 v1의 "비언어적 태도 20%"는 제거하고, 전달력(25→35%)과 안정성(15→25%)으로 재분배해 발화 코칭에 집중. (P2 영상 분석 도입 시 별도 보조 점수로 표기하되 종합 점수에는 미반영)

### 10.2 세부 점수

내용 점수 / 전달력 점수 / 안정성 점수 (자세 점수는 P2에서 보조 표기)

## 12. 비기능 요구사항

| 항목          | 요구사항                                                     |
| ------------- | ------------------------------------------------------------ |
| 응답성        | 분석 요청 후 진행 상태(로딩)를 표시해야 한다.                |

| 개인정보      | MVP에서는 파일·결과를 장기 저장하지 않는다.                  |
| 확장성        | DB·로그인·영상 분석을 붙일 수 있도록 API를 분리한다.         |

## 16. 최종 요약

Speech Mirror(말거울)는 로그인 없이 쓰는 웹 기반 AI 발화 코칭 모의면접 서비스다. 사용자는 이력서·직종을 입력하고 연습/실전 모드로 면접을 진행한다. 종료 후 시스템은 답변 오디오를 분석해 전사 텍스트, 말 속도, 침묵, **한국어 필러 단어**, 답변 내용, 전체 점수를 제공한다. 영어 중심 글로벌 도구나 내용 중심 국내 모의면접과 달리, **한국어 발화 그 자체를 교정**하는 데 집중한다. 영상/자세 분석은 데모 안정성을 위해 P2 확장으로 분리한다.
