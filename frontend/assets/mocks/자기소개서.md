## 지원동기

**"불편함을 해결하는 기술"의 가치를 직접 경험했습니다.**

교회 웹사이트를 개발하며 깨달았습니다. 흩어져 있던 설교 영상, 주보, 교회 소식을 한 곳에 모으니 교인들이 시간과 장소에 구애받지 않고 정보를 얻을 수 있게 되었습니다. 기술은 단순히 기능을 만드는 것이 아니라, 일상의 불편함을 해결하는 도구라는 것을 체감한 순간이었습니다.

커버링의 "쓰레기로 고민하지 않는 세상"이라는 미션을 접했을 때, 같은 방향을 바라보고 있다고 느꼈습니다. 새벽 배송과 간편 결제는 혁신했지만, 쓰레기 처리는 여전히 정해진 요일, 정해진 장소, 정해진 방식을 따라야 합니다. 이 불편함을 기술로 해결하겠다는 커버링의 접근 방식에 깊이 공감했습니다.

"빨리 실패할 용기를 가집니다"라는 커버링의 문화는 제 "일단 해보자!"라는 모토와 맞닿아 있습니다. 개발을 시작한 지 1년이지만, 실패를 두려워하지 않고 Flutter 앱 2개를 양대 스토어에 출시했고, 그 외에도 팀 프로젝트와 개인 프로젝트를 통해 다양한 Flutter 앱을 개발해왔습니다. 이 경험과 실행력으로 커버링 라이더스 앱 고도화에 기여하고 싶습니다.

무엇보다 Flutter로 실서비스 개발에 참여할 수 있다는 점이 가장 큰 지원 동기입니다. 개인 프로젝트를 넘어 실제 사용자에게 영향을 주는 서비스를 경험하고 싶습니다.

## 직무 관련 경험

**1년간 다양한 프로젝트, Flutter 앱 출시부터 웹 운영까지**

개발을 시작한 지 1년이 되어갑니다. 짧은 시간이지만 "일단 만들어보자"는 마음으로 다양한 도전을 했습니다.

- **[Flutter 앱 개발 & 출시]** RuleBook과 Block Drag를 개발하여 Google Play Store와 App Store에 출시했습니다. Provider에서 Riverpod + Freezed로 상태관리를 개선하고, Clean Architecture를 적용해 확장성 있는 구조를 설계했습니다. Block Drag는 사용자 피드백을 반영해 4회 이상 디자인과 게임 기능을 추가 및 수정하며, 기대를 뛰어넘는 수준을 추구했습니다.

- **[풀스택 개발 경험]** 은샘교회 웹사이트를 Next.js + FastAPI로 5개월간 개발했습니다. 프론트엔드부터 백엔드 API 설계, CMS 시스템 구축까지 전 과정을 경험했습니다. 이 경험을 바탕으로 NestJS 백엔드 학습에도 빠르게 적응할 자신이 있습니다.

- **[성능 최적화]** 금융 교육 앱에서 퀴즈 시스템의 API 호출을 32회에서 2회로 94% 감소시켰고, 검색 기능에 디바운싱을 적용해 불필요한 호출을 90% 줄였습니다. 하면 좋을 10가지보다 임팩트 하나에 집중하는 경험이었습니다.

- **[협업 경험]** 6인 팀 프로젝트에서 Notion으로 PRD/아키텍처를 문서화하고, GitHub Issue로 작업을 관리하며, Figma 디자인을 기반으로 개발했습니다. 필요한 말을 돌리지 않고 직접 전달하며 협업하는 방법을 익혔습니다.

- **[자동화 경험]** 반복되는 작업은 자동화해서 효율적으로 일하려고 합니다. GitHub Actions로 CI/CD 파이프라인을 구축해 빌드와 배포를 자동화한 경험이 있습니다.

## 문제 해결 및 도전 경험

**40kg 감량이 알려준 것: 도전은 습관이 된다**

개발을 시작하기 전, 저는 40kg을 감량한 경험이 있습니다. 처음엔 막막했지만, 매일 조금씩 실천하다 보니 불가능해 보이던 목표가 현실이 되었습니다. 이 경험은 "시작하면 된다"는 확신을 심어주었고, 개발에서도 같은 태도로 임하게 되었습니다.

- **[도전: 해본 적 없는 앱 출시]** RuleBook을 개발할 때 앱 개발 경험이 전무했습니다. 하지만 교회에서 출석 관리가 수기로 이뤄지며 생기는 불편함을 해결하고 싶었습니다. 4개월간 Flutter를 독학하며 개발했고, 결국 양대 스토어 출시에 성공했습니다. 바퀴 없는 멋진 자동차가 아닌, 바로 탈 수 있는 퀵보드를 만든다는 마음으로 MVP를 완성했습니다.

- **[문제 해결: 구조적 한계 극복]** Block Drag는 RuleBook 직후에 진행한 두 번째 앱이었습니다. 경험이 부족하다 보니 초기 버전은 구조가 체계적이지 못하고 확장성도 부족했습니다. 이를 인지하고 종강 후 2개월간 전면 리팩토링을 진행했습니다. Clean Architecture를 적용하고, 상태관리를 Riverpod + Freezed로 개선했습니다. 실패해도 더 나은 보완책을 고민하고 끝까지 개선해 결과를 만드는 경험이었습니다.

- **[낯선 영역 도전]** Tripgether에서 SNS 링크 공유 기능을 구현할 때, iOS Share Extension과 Android Intent Filter라는 처음 접하는 네이티브 영역에 부딪혔습니다. 공식 문서와 다양한 개발 커뮤니티를 활용해 해결했고, 이 경험으로 익숙한 틀을 벗어나 과감하게 시도하는 것의 가치를 체감했습니다.

## 기대 성과 목표

**초반: 빠르게 적응하고 기여 시작하기**

Flutter로 앱 2개를 양대 스토어에 출시하고 Clean Architecture와 상태관리를 경험했기에, 커버링 라이더스 앱의 코드베이스에 빠르게 적응할 자신이 있습니다. 인턴십 초반에는 서비스 도메인 구조와 서버-클라이언트 데이터 흐름을 파악하고, 기존 앱 기능 개선과 에러 처리 로직 개선부터 차근차근 기여해나가겠습니다.

**중반: 실서비스에 임팩트 있는 기여하기**

신규 기능 설계부터 구현까지, 단순히 시키는 일만 하는 것이 아니라 사용자 니즈 기반으로 요구사항을 정의하고 주도적으로 제안하겠습니다. NestJS 백엔드도 함께 경험하며 프론트엔드와 백엔드를 아우르는 풀스택 관점을 익히겠습니다. PR 리뷰와 QA를 거쳐 실제 서비스에 배포되는 전체 개발 사이클을 경험하고 싶습니다.

**인턴십 이후: 커버링과 함께 성장하는 개발자로**

인턴십을 통해 Flutter 실서비스 개발 역량과 백엔드 기본기를 갖추겠습니다. 정규직 전환 기회가 주어진다면, 커버링 라이더스뿐만 아니라 커버링의 다양한 서비스에서 "쓰레기로 고민하지 않는 세상"이라는 미션처럼, 일상의 불편함을 기술로 해결하는 개발자가 되고 싶습니다.
