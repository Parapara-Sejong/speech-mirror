# Pricing Page UI — 설계

- 날짜: 2026-06-24
- 범위: UI 전용 가격 화면

## Context

Speech Mirror에는 로그인/결제 기능이 없다. 이번 작업의 목표는 실제 결제 플로우가 아니라, 서비스가 어떤 방식으로 수익화될 수 있는지 보여주는 가격 화면을 만드는 것이다.

## 결정

- `/pricing` 단독 페이지를 추가한다.
- 기존 `PRICING_TIERS`와 `PricingTierCard`를 그대로 재사용한다.
- showcase에서 검증된 `HeroBand`, `CtaBand`, `Footer`, `Button` 조합을 사용한다.
- 결제, 로그인, API 연동, 새 상태 관리는 만들지 않는다.

## UI 구성

1. 상단 히어로: 수익화 구조를 설명하는 제목과 보조 문구.
2. 가격 카드 3개: 무료/프로/맥스 티어를 반응형 그리드로 노출.
3. 안내 밴드: 현재는 결제 연결 없는 예시 가격이라는 점을 UI 문구로 표시.
4. Footer: 기존 섹션 컴포넌트 재사용.

## 검증

- `npm run build`로 타입체크와 Vite 빌드가 통과해야 한다.
- `/pricing` 라우트가 렌더링되고 기존 `/showcase`는 깨지지 않아야 한다.
