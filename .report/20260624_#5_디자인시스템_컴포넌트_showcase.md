# DESIGN.md 기반 디자인 시스템 컴포넌트 + Showcase 구축

### 📌 작업 개요
DESIGN.md(cream + coral + dark navy)의 디자인 시스템을 React 프론트엔드의 재사용 컴포넌트로 구현. Tailwind v4 `@theme` 토큰화 + UI/섹션 컴포넌트 세트 + 이를 한눈에 검수하는 `/showcase` 갤러리 페이지 추가. 제품 맥락은 AI 모의 면접, 가격 티어(무료/프로/맥스) 포함.

### 🎯 구현 목표
- DESIGN.md의 색·타이포·radius·spacing을 Tailwind 토큰으로 일원화
- 버튼/카드/배지/탭/입력/섹션 등 재사용 컴포넌트 구축
- 기존 analysis 데모와 분리된 showcase 페이지를 라우팅으로 제공

### ✅ 구현 내용

#### 디자인 토큰 · 폰트 · 유틸 (기반)
- **파일**: `src/styles.css`, `index.html`, `src/lib/cn.ts`, `package.json`
- **변경 내용**: `@theme`에 DESIGN.md 색/타이포(size·line-height·letter-spacing)/radius/`spacing-section` 토큰 정의 → `bg-canvas`·`text-display-xl`·`rounded-lg` 등 전역 유틸 생성. Pretendard + JetBrains Mono를 `<link>`로 로드. 클래스 결합용 `cn` 유틸 추가. `react-router-dom` 의존성 추가
- **이유**: 토큰 단일화로 컴포넌트 전반의 색·간격·타이포 일관성 확보

#### UI 컴포넌트 (`src/components/ui/`)
- **파일**: `Button`, `IconButton`, `Badge`, `CategoryTab`, `TextInput`, `FeatureCard`, `ProductMockupCardDark`, `CodeWindowCard`, `CalloutCardCoral`, `PricingTierCard`
- **변경 내용**: 1파일 1컴포넌트, 토큰 유틸 기반 스타일. Button은 4 variant(primary/secondary/secondary-on-dark/text-link), PricingTierCard는 featured 시 dark surface 반전
- **이유**: DESIGN.md 컴포넌트 스펙을 조합 가능한 형태로 정리

#### 섹션 컴포넌트 (`src/components/sections/`)
- **파일**: `HeroBand`, `CtaBand`, `Footer`
- **변경 내용**: HeroBand 6/6 반응형 그리드, CtaBand coral/dark variant, Footer 4열→1열 반응형 + 기본 컬럼
- **이유**: 페이지 레이아웃 단위 재사용

#### 가격 데이터
- **파일**: `src/constants/pricing.ts`
- **변경 내용**: `PricingTier` 타입 + `PRICING_TIERS`(무료/프로/맥스, 더미 가격·기능). 프로 = featured
- **이유**: 콘텐츠를 한 파일로 분리해 추후 수정 용이(calibration 노브)

#### 라우팅 · 페이지 분리
- **파일**: `src/App.tsx`, `src/pages/AnalysisPage.tsx`, `src/pages/ShowcasePage.tsx`
- **변경 내용**: App을 `BrowserRouter` 셸로 축소. 기존 App 본문을 `AnalysisPage`(dark, `/`)로 이동, `ShowcasePage`(cream, `/showcase`)에서 전체 컴포넌트를 카테고리별로 나열 + 가격 3종
- **이유**: analysis 데모와 디자인 검수 화면을 분리 제공

#### 가격 카드 CTA 위치 통일
- **파일**: `src/components/ui/PricingTierCard.tsx`
- **변경 내용**: 레이아웃을 `이름 → 가격 → CTA → 구분선 → 기능목록` 순으로 변경. 기능 개수와 무관하게 CTA 버튼 정렬
- **이유**: 레퍼런스 디자인과 동일한 통일성 확보

### 🔧 주요 변경사항 상세

#### Tailwind v4 토큰 네이밍
`--color-body`(텍스트 색)와 `--text-body-md`(폰트 크기)가 `text-*` 네임스페이스를 공유하나, 접미사가 달라 충돌 없이 공존. display 위계는 serif 대신 Pretendard weight(`font-semibold`)로 보강

**특이사항**:
- DESIGN.md의 Copernicus/StyreneB는 라이선스 폰트라 한글 커버리지 가능한 **Pretendard로 통일**(serif 미사용)
- 해커톤 진행으로 **자동 테스트(vitest) 제외** — 검증은 `npm run build` + `/showcase` 육안

### 📦 의존성 변경
