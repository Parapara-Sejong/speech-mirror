## 제목

⚙️ [기능추가][백엔드][FastAPI] 이력서·자소서·인재상 기반 GEMINI 질문 생성 및 발화 분석 API 구축

## 본문

## 📝 현재 문제점

- 프론트는 진입부터 결과까지 흐름이 임시(mock) 시임으로 완성됐지만, 백엔드가 없어 실제 질문 생성과 발화 분석이 동작하지 않는다.
- 이력서·자기소개서·인재상 파일에서 텍스트를 추출해 맞춤 질문을 만드는 기능, 답변 오디오를 분석해 리포트를 만드는 기능이 비어 있다.
- 데모에서 진짜 입력으로 질문이 생성되고 결과가 나오는 흐름을 보여줄 수 없다.

## 🛠️ 해결 방안 / 제안 기능

- 이 레포 backend/ 에 FastAPI 백엔드를 구축한다.
- 이력서·자기소개서·인재상(파일 또는 텍스트)과 직종·면접 종류·인재상 키워드를 받아 GEMINI로 면접 질문을 생성한다. 각 질문에 평가 역량 태그를 포함한다.
- 답변 오디오를 받아 발화 분석 리포트(전사·말 속도·침묵·필러·내용 피드백·점수)를 제공한다.
- 프론트의 기존 mock 시임(질문 생성, 결과 조회)을 실제 API 호출로 교체해 연결한다.
- 범위 밖(후순위): 실제 STT 정밀도 튜닝, 영상 분석, 다중 사용자 세션.

## ⚙️ 작업 내용

- FastAPI 프로젝트 뼈대(backend/) 구성. 서버 포트 8080, 프론트(localhost:5173·5175) CORS 허용.
- 질문 생성 API(POST /questions/recommend): 이력서·자기소개서·인재상 파일/텍스트 + 직종·면접 종류·인재상 키워드 입력 → 평가 역량 태그가 포함된 추천 질문 목록 반환.
- 파일 텍스트 추출(PDF·DOCX·MD·TXT)은 백엔드에서 처리(프론트는 원본 파일 업로드만).
- GEMINI 연동. API 키는 서버 환경변수(GEMINI_API_KEY)로 관리하고 프론트에 노출하지 않는다.
- 발화 분석 API: 답변 오디오 업로드 → 분석 실행 → 결과 조회(PRD §8.2, §11 계약 준수). 진행 상태 폴링 지원.
- 프론트 시임 교체 연결: 질문 생성(questionApi.ts), 결과 조회(useAnalysisReportQuery.ts).
- 우선순위: 1차 질문 생성 API, 2차 발화 분석 API.
