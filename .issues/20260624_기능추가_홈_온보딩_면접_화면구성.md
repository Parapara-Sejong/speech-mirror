## 제목

⚙️ [기능추가][프론트][화면구성] 홈·온보딩·질문선택·면접 화면 구성 (남은 핵심 흐름)

## 본문

## 📝 현재 문제점

- 결과 화면은 구현되었으나 그 앞단(진입부터 면접까지)이 비어 있어 사용자가 처음부터 끝까지 흐름을 탈 수 없다.
- 홈, 이력서 업로드, 직종/면접 종류/모드 선택, 질문 선택, 면접 진행 화면이 없어 결과 화면만 단독으로 떠 있는 상태다.
- 데모에서 진입부터 결과까지 하나의 흐름으로 보여줄 수 없다.

## 🛠️ 해결 방안 / 제안 기능

- PRD 사용자 시나리오(5장)에 맞춰 남은 화면들을 구성한다.
- 백엔드 연동 없이 임시 데이터로 화면·라우팅·상태 전환을 먼저 완성해, 진입부터 결과까지 한 흐름으로 시연 가능하게 한다.
- 결과 화면과 동일한 디자인 시스템(토큰)으로 일관성을 유지한다.
- 시간 제약으로 이슈는 하나로 묶되, 화면 단위 슬라이스로 나눠 순차 구현한다.

## ⚙️ 작업 내용

- 홈/랜딩 화면: 서비스 소개와 시작 진입
- 이력서 업로드: 파일 업로드와 텍스트 추출 입력, 미지원 형식/추출 실패 시 직접 입력 대체
- 직종·면접 종류·연습/실전 모드 선택
- 질문 추천·선택: 추천 질문 표시(평가 역량 태그 포함), 연습 모드 3개 선택 + 랜덤 1개 추가로 4문항 구성
- 면접 진행: 현재 질문 표시, 마이크 인식 상태 표시, 답변 오디오 녹음, 분석하기 진입
- 화면 간 라우팅과 진행 상태 연결, 결과 화면으로 연결
- 범위 밖(별도/후순위): 실제 STT·백엔드 연동, 카메라/시선 등 영상 비언어 지표, 영상 다시보기
