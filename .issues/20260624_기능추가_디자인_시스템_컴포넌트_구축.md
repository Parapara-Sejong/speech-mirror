## 본문

## 📝 현재 문제점

- React 프론트엔드에 DESIGN.md에서 정의한 디자인 시스템을 재사용 가능한 컴포넌트로 표현할 구조가 아직 부족합니다.
- 현재 화면은 speech analysis 결과 조회 흐름 중심으로 구성되어 있어, warm cream canvas, coral CTA, dark product surface, feature card 등 공통 UI 패턴을 일관되게 확장하기 어렵습니다.
- 향후 페이지나 기능을 추가할 때 동일한 색상, 타이포그래피, spacing, card, button 규칙을 반복해서 맞춰야 할 가능성이 있습니다.

## 🛠️ 해결 방안 / 제안 기능

- DESIGN.md를 기준으로 React 프로젝트에서 재사용 가능한 UI 컴포넌트 세트를 구축합니다.
- Claude 스타일의 핵심 분위기인 cream canvas, coral primary action, dark product mockup surface, light cream card 리듬을 화면 구성에 반영할 수 있게 합니다.
- 버튼, 카드, 배지, 탭, 입력, 섹션 레이아웃, CTA band, product mockup surface 등 기본 컴포넌트를 페이지에서 조합할 수 있는 형태로 정리합니다.
- 컴포넌트가 모바일과 데스크톱에서 모두 자연스럽게 표시되도록 responsive layout 기준을 함께 반영합니다.

## ⚙️ 작업 내용

- DESIGN.md의 color, typography, spacing, radius 기준을 프론트엔드 스타일 규칙으로 반영
- primary, secondary, text-link, circular icon button 등 버튼 컴포넌트 구성
- feature card, product mockup dark card, code window card, callout card 등 카드 컴포넌트 구성
- badge, category tab, text input 등 기본 UI 컴포넌트 구성
- hero band, CTA band, footer 등 섹션 레이아웃 컴포넌트 구성
- 기존 Speech Mirror 화면에서 새 컴포넌트를 적용할 수 있는 진입 지점 정리
- DESIGN.md의 Do/Don't 기준에 맞춰 색상, 여백, 타이포그래피 일관성 확인
